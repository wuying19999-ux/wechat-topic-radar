# 05 Bug 记录规范

## 记录位置

优先在管理员页“试运营测试与 Bug 看板”记录。

也可以在 GitHub Issue 里使用模板：

```text
.github/ISSUE_TEMPLATE/bug_report.md
```

## Bug 状态

- 待处理：已发现，还没开始修。
- 处理中：正在定位或修复。
- 已修复：已上线或已验证。
- 暂缓：暂时不影响试运营，后续再处理。

## 严重程度

- 轻微：不影响核心使用。
- 一般：影响体验，但有绕过方式。
- 严重：影响生成、保存或主要流程。
- 阻塞：员工无法继续使用。
