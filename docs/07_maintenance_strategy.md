# 07 后续维护策略

## 资料维护

当前资料重点覆盖 5-8 月新生开学前场景，后续需要补齐 9 月到次年 4 月：

- 开学后课程适应。
- 作业、论文、presentation。
- 考试周和补考。
- 二手群持续活跃。
- 学术支持和安全提醒。

## 产品植入规划

开学季后，群话题可以自然延展到公司产品：

- 上课听不懂：引导到学习辅导和课程支持。
- 作业不会写：引导到合规写作指导和引用规范说明。
- AI 使用焦虑：引导到 AI 安全与学术诚信提醒。
- 校园安全、诈骗、二手交易：引导到安全 AI 或风险提醒服务。

所有植入必须以真实痛点和群友问题为起点，避免硬广。

## 运营复盘

每周查看：

- 发布次数。
- 回复数。
- 活跃效果。
- 质量评分。
- Bug 数和修复数。
- 高风险内容记录。

## 技术维护

- 定期检查 Vercel 环境变量是否有效。
- 定期检查 Supabase 表是否写入正常。
- API 不稳定时优先保证本地资料库兜底。
- 新增学校资料后重新导入知识库并部署。
