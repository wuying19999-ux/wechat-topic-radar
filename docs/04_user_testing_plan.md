# 04 试运营测试计划

## 测试目标

验证员工是否能用网页更快生成可发布内容，并记录发布后的群活跃效果。

## 建议测试周期

至少 3 天，每天安排不同员工测试不同学校和群模块。

## 每日记录项

- 测试人。
- 测试日期。
- 学校。
- 模块。
- 生成次数。
- 发布次数。
- 发现 Bug 数。
- 备注。

## 活跃效果记录项

员工发布具体对话后，在“记录试运营”里填写：

- 员工/账号。
- 发布群名。
- 群人数。
- 发布时间。
- 回复数。
- 活跃效果：好 / 一般 / 差。
- 修改程度：直接使用 / 小改后使用 / 大改后使用 / 未使用。
- 质量评分。
- 风险情况。
- 补充备注。

## 数据查看

管理员进入：

```text
https://wechat-topic-radar.vercel.app/?admin=1
```

可以查看试运营汇总、测试记录和 Bug 看板。
